# Agent-FPL

A specialized agentic AI framework built to automate data analysis and dominate the Fantasy Premier League (FPL). The system provides AI-powered transfer suggestions and squad building recommendations by combining live FPL data with a multi-agent LLM pipeline.

---

## Architecture

```
Agent-FPL/
├── etl/                  # Data pipeline — fetches and transforms FPL data
├── Core/                 # Agent logic, LLM configurations, sub-agents
├── Web/
│   ├── webservice/       # FastAPI backend
│   └── frontend/         # Flask frontend
├── Utils/                # Shared utilities (Spark, logging, env)
├── Constants/            # Column definitions and data constants
└── data/                 # Parquet data files (generated by ETL)
```

## Setup

### Prerequisites

- Python 3.10+
- Java 8+ (required for PySpark)
- A Hugging Face account and API token

### Install dependencies

```bash
pip install -r requirements.txt
```

### Environment variables

Rename `.env.sample` to `.env` in the project root and fill in the values:

```env
# Environment
APP_ENV=dev                         # dev or production

# API
API_HOST=127.0.0.1                  # local IP, private IP, or domain

# Hugging Face Inference API
HF_TOKEN=hf_xxxxxxxxxxxxxxxxxxxx

# Log directories
ETL_LOGS_DIR=logs/etl_logs
API_LOGS_DIR=logs/webservice_logs
FRONTEND_LOGS_DIR=logs/frontend_logs

# Data directory
DATA_DIR=data
```

## Running the System

### 1. Load data (ETL)

Run once before starting the application. Fetches player, team and position data from the FPL API and saves as Parquet files.

```bash
python3 -m etl.main
```

> Re-run at the start of each gameweek to keep data up to date.

### 2. Start the application

```bash
bash start.sh
```

This starts all services. Once running:

| Service | URL |
|---------|-----|
| Frontend | `http://{API_HOST}:5000` |
| Webservice | `http://{API_HOST}:8000` |
| API Docs | `http://{API_HOST}:8000/docs` |

To use a custom env file:

```bash
bash start.sh .env.production
```

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/get_manager_details/{manager_id}` | Fetch FPL manager profile |
| `GET` | `/get_manger_players/{manager_id}` | Fetch manager's current squad (requires `Authorization` header) |
| `POST` | `/get_player_suggestion/{manager_id}` | Get AI transfer suggestion for a selected player |
| `GET` | `/get_squad_suggestion/{manager_id}` | Get AI squad building recommendation |

### Authentication

The `/get_manger_players` endpoint requires your FPL session token passed as a request header:

```
Authorization: <token>
```

Obtain this from your browser's DevTools → Network tab → any FPL API request → `Authorization` header.

---

## Privacy

This application does not store any personal information. Your Manager ID and Auth Token are used only for the duration of your session and are never saved, logged, or shared. This includes sensitive account data such as your email address that may be accessible via the FPL API.

---

## How it works

1. **ETL** pulls all player, team and position data from the public FPL API and stores it locally as Parquet files.
2. **FastAPI** joins your live squad (fetched using your auth token) against the local Parquet data using PySpark.
3. **LLM agents** (via Hugging Face Inference API) analyse the enriched player data and generate transfer or squad suggestions.
4. **Flask frontend** presents your squad and streams agent suggestions back without exposing the backend API to the browser.
